# Tools (Engineering + MLOps)

This track is about shipping and evaluating ML/LLM systems safely and repeatably.

## What to Learn

### Foundations
- Git + GitHub workflow (issues, PRs, reviews)
- Python environments (venv/conda), dependency management
- Notebooks vs scripts (when to use each)

### Build + Ship
- FastAPI (simple model/LLM APIs)
- Docker (reproducible runtime)
- Basic CI (lint/test on PR)

### LLM App Tooling
- RAG basics (chunking, retrieval, reranking, evaluation)
- Serving options (vLLM / llama.cpp / hosted endpoints)
- Logging + redaction (don’t leak secrets)

### Experiment Tracking (optional but useful)
- MLflow or Weights & Biases

### AI Coding Assistants (workflow)
- Codex: refactors, docs, tests, small prototypes
- Claude Code: repo navigation, drafts, quick iterations
- "AntiGravity": treat prompts/evals as versioned artifacts (repeatable runs)

## Recommended Learning Order
1) Git + Python envs
2) FastAPI + Docker
3) Basic CI + testing
4) RAG + eval harnesses
5) Serving + observability
