# Join Neuralchemy (Academy / Research)

This repo is the public learning + skills hub for Neuralchemy Academy. If you want to collaborate, contribute, or join the team, use this doc as the starting point.

## Ways to Join

### 1) AI Security Researcher (LLM / Agent Security)
Good fit if you can do some of:
- Threat modeling for LLM apps (RAG, tools, agents)
- Prompt-injection + data exfiltration research
- Model evaluation, jailbreak testing, red teaming
- Secure-by-design guidance for AI product teams

What to contribute here:
- A short writeup in `Ai-security/` (attacks, defenses, eval checklists)
- A small demo project in `Projects/` (repro + mitigations)

### 2) ML Researcher (Applied / Systems)
Good fit if you can do some of:
- Training/evaluation workflows, ablations, and metrics
- Data pipelines, labeling strategy, dataset QA
- Fine-tuning, distillation, retrieval, or reasoning evals

What to contribute here:
- Improve track READMEs with better learning sequences
- Add proof-of-skill projects + rubrics in `Projects/`

### 3) Curriculum / Track Builder
Good fit if you can:
- Organize learning paths and prerequisites
- Turn link dumps into coherent "do this -> then this" tracks

What to contribute here:
- Rewrite/structure one track README (keep it concise + actionable)
- Add milestones + recommended projects

### 4) Maintainer / Community Builder
Good fit if you can:
- Review PRs, fix broken links, keep structure consistent
- Triage issues, label requests, keep things moving

## How to Apply (lightweight)
Open a GitHub issue titled: "Join: <role> - <your name/handle>" with:
- Your role(s): AI Security / ML Research / Curriculum / Maintainer
- Links: GitHub, blog, papers, repos (anything relevant)
- What you want to work on in this repo (1-3 bullets)
- Weekly bandwidth (rough estimate)

If you prefer a PR-first approach: submit one small PR (fix links, add a project, improve a section) and mention you want to join in the PR description.

## Quality Bar
- Prefer clarity over quantity.
- For security content: include a threat model + reproduction steps + mitigations.
- For learning content: include prerequisites, milestones, and "build this" projects.
